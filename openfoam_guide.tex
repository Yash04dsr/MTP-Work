\chapter{Introduction}
\label{ch:intro}

This guide documents the complete OpenFOAM setup for a transient
Delayed Detached Eddy Simulation (DDES) of hydrogen injection into a
methane-filled T-junction pipeline.  Every dictionary file, boundary
condition, and solver setting is a direct translation of the validated
ANSYS Fluent configuration described in the companion
\emph{Parameterised CFD Simulation Guide}.

\section{Simulation Overview}

\begin{itemize}
  \item \textbf{Physics:} Compressible binary-species mixing
        (H$_2$\,/\,CH$_4$), no combustion.
  \item \textbf{Turbulence:} Spalart--Allmaras DDES with curvature
        correction.
  \item \textbf{Operating pressure:} 6.9\,MPa.
  \item \textbf{Solver:} \texttt{reactingFoam} (compressible,
        multi-species, transient).
  \item \textbf{Time step:} $\Delta t = 2\times10^{-4}$\,s,
        25\,000 steps (5\,s physical time).
\end{itemize}

\section{Step 1: Generate the STL}

A Python script (\texttt{generateSTL.py}) creates the T-junction
surface as four separate STL files in \texttt{constant/triSurface/}:

\begin{enumerate}
  \item \texttt{wall.stl} --- combined wall surfaces of both pipes.
  \item \texttt{main\_inlet.stl} --- cap at $Z = 0$.
  \item \texttt{outlet.stl} --- cap at $Z = 9.2$\,m.
  \item \texttt{branch\_inlet.stl} --- cap at the branch tip.
\end{enumerate}

\noindent Run:
\begin{lstlisting}[language=bash,numbers=none]
python3 generateSTL.py
\end{lstlisting}

\noindent\textbf{Dependency:} \texttt{numpy} and \texttt{numpy-stl}
(\texttt{pip install numpy-stl}).

\section{Step 4: \texttt{snappyHexMesh}}

The \texttt{snappyHexMeshDict} performs three operations:

\subsection{Castellated Mesh}

\begin{itemize}
  \item Refines cells intersecting the STL surfaces to \textbf{level 2--3}.
  \item Applies \textbf{level 3} refinement inside a sphere of radius
        690\,mm centred at the junction $(0, 0, 4.6)$ --- matching the
        Fluent BOI region ($1.5 \times d_1$).
  \item Removes cells outside the pipe geometry.
  \item \texttt{locationInMesh}: $(0, 0, 2.0)$ --- a point guaranteed
        to be inside the main pipe.
\end{itemize}

\subsection{Snapping}

Mesh vertices are iteratively moved onto the STL surface.  The feature
edges (\texttt{.eMesh} files) ensure clean resolution of the pipe--branch
intersection.

\subsection{Boundary Layers}

Five prismatic layers are added to the \texttt{wall} patch:

\begin{table}[H]
\centering
\caption{Layer insertion parameters.}
\begin{tabular}{@{}lr@{}}
\toprule
\textbf{Parameter}     & \textbf{Value} \\
\midrule

Final layer thickness   & 0.3 (relative) \\
Min thickness           & 0.1 (relative) \\

\chapter{Physical Models and Turbulence}
\label{ch:models}

\section{Turbulence Model (\texttt{momentumTransport})}

The turbulence model is configured in
\texttt{constant/momentumTransport}:

\begin{lstlisting}[language={},title=constant/momentumTransport]
simulationType  LES;

LES
{
    LESModel        SpalartAllmarasDDES;
    turbulence      on;
    printCoeffs     on;
    delta           cubeRootVol;

    cubeRootVolCoeffs
    {
        deltaCoeff  1;
    }

    SpalartAllmarasDDESCoeffs
    {
        curvatureCorrection yes;
    }
}
\end{lstlisting}

\subsection{Key Settings}

\begin{itemize}
  \item \textbf{simulationType LES:}  In OpenFOAM, DES/DDES models
        are classified under the LES framework.  The
        \texttt{SpalartAllmarasDDES} model automatically switches
        between RANS behaviour in attached boundary layers and LES
        behaviour in separated regions.
  \item \textbf{Curvature correction:}  Enabled to improve prediction
        accuracy in the curved flow region near the T-junction.
        Equivalent to the ``Curvature Correction'' checkbox in
        Fluent's viscous model panel.
  \item \textbf{delta cubeRootVol:}  The LES filter width is taken as
        the cube root of the cell volume ($\Delta = V^{1/3}$), which
        is the standard choice for unstructured meshes.
\end{itemize}

\subsection{Turbulent Schmidt Number}

The turbulent Schmidt number is set in
\texttt{constant/turbulenceProperties}:

\begin{lstlisting}[language={},title=constant/turbulenceProperties]
Sct     0.5;
\end{lstlisting}

\noindent This matches the Fluent TUI command
\texttt{/define/models/viscous/turbulent-schmidt-number 0.5}.

\section{Gravity}

Gravity is set in \texttt{constant/g}:

\begin{lstlisting}[language={},title=constant/g]

value       (0 -9.81 0);
\end{lstlisting}

\noindent Combined with the \texttt{perfectGas} equation of state,
the solver fully resolves buoyancy-driven stratification (density
ratio CH$_4$:H$_2$ $\approx 8$ at 6.9\,MPa).  This is equivalent
to Fluent's gravity enablement with \emph{Specified Operating
Density = 0}.

\subsection{Branch Velocity Derivation}

The branch velocity of 32\,m/s is derived from the volumetric
dilution factor $\mathcal{D} = 0.2$:
\[
  \mathcal{D} = \beta^2 \frac{u_2}{u_1}
  \quad\Rightarrow\quad
  0.2 = (0.25)^2 \times \frac{u_2}{10}
  \quad\Rightarrow\quad
  u_2 = 32\;\text{m/s}
\]

{

    nCorrectors         2;

}
\end{lstlisting}

\section{Discretisation Schemes (\texttt{fvSchemes})}

\subsection{Temporal}

\begin{lstlisting}[language={},numbers=none]

\end{lstlisting}

\noindent \texttt{backward} is second-order implicit and unbounded.
It is the OpenFOAM equivalent of Fluent's \textbf{Bounded Second

\subsection{Gradient}

\begin{lstlisting}[language={},numbers=none]
gradSchemes { default  Gauss linear; }
\end{lstlisting}

\noindent Equivalent to Fluent's \textbf{Least Squares Cell Based}
gradient.

turbulent structures in DES regions while maintaining stability.
This is the closest OpenFOAM equivalent to Fluent's Bounded Central
Differencing.}}

\subsection{Laplacian and Surface-Normal Gradient}

\begin{lstlisting}[language={},numbers=none]
laplacianSchemes { default  Gauss linear corrected; }

\end{lstlisting}

\noindent The \texttt{corrected} scheme provides second-order accuracy
on non-orthogonal meshes.  This handles the equivalent of Fluent's
\textbf{PRESTO!} pressure interpolation via the corrected Laplacian.

\section{Step 6: During the Run --- What to Watch}

\begin{enumerate}
  \item \textbf{Residuals:}  Check
        \texttt{postProcessing/residuals/} --- all residuals should
        drop by at least 1--2 orders of magnitude within each time
        step.
  \item \textbf{Mass balance:}  Compare inlet and outlet mass flow
        rates.  Imbalance $> 1\,\%$ indicates a problem.
  \item \textbf{Outlet H$_2$:}  Should stabilise near $\sim 0.149$
        for $\mathcal{D} = 0.2$.
  \item \textbf{Courant number:}  The solver prints the maximum
        Courant number each time step.  Target: $\text{Co} < 2$.
        If consistently higher, reduce $\Delta t$ to $1\times10^{-4}$\,s.
\end{enumerate}


\section{Step 7: Enable Time Statistics (After $t = 4$\,s)}

The \texttt{fieldAverage1} function object in \texttt{controlDict}
is pre-configured to begin time-averaging at $t = 4$\,s
(\texttt{timeStart 4.0}).  The first 20\,000 steps (4\,s) are
treated as the initial transient; the final 5\,000 steps (1\,s)
are used for time-averaged statistics.

Fields averaged:
\begin{itemize}
  \item \texttt{UMean}, \texttt{UPrime2Mean} ---
        mean velocity and Reynolds stresses.
  \item \texttt{pMean} --- mean pressure.
  \item \texttt{H2Mean}, \texttt{H2Prime2Mean} ---
        mean H$_2$ concentration and its variance (for COV calculation).
  \item \texttt{TMean} --- mean temperature.
\end{itemize}

\chapter{Post-Processing}
\label{ch:post}

\section{Hydrogen Volume Fraction}

Fluent solves for mass fraction $Y_{\mathrm{H_2}}$, while the
reference paper reports volume fraction $\phi_{\mathrm{H_2}}$.
The conversion is:

\begin{equation}
  \phi_{\mathrm{H_2}}
  = \frac{Y_{\mathrm{H_2}} / M_{\mathrm{H_2}}}
         {Y_{\mathrm{H_2}} / M_{\mathrm{H_2}}
          + (1 - Y_{\mathrm{H_2}}) / M_{\mathrm{CH_4}}}
  \label{eq:volfrac}
\end{equation}

\noindent where $M_{\mathrm{H_2}} = 2.016$\,kg/kmol and
$M_{\mathrm{CH_4}} = 16.043$\,kg/kmol.

In OpenFOAM, compute this in ParaView using a \textbf{Calculator}
filter:

\begin{lstlisting}[language={},numbers=none]
(H2/2.016) / (H2/2.016 + (1-H2)/16.043)
\end{lstlisting}

\noindent Alternatively, add a \texttt{coded} function object to
\texttt{controlDict} for automatic computation during the run.


\section{Visualisation in ParaView}

Open the reconstructed case:

\begin{lstlisting}[language=bash,numbers=none]
paraFoam
\end{lstlisting}

\noindent Key visualisations:

\begin{enumerate}
  \item \textbf{Contour plots:}  H$_2$ mass/volume fraction on
        cross-sectional planes downstream of the junction.
  \item \textbf{Streamlines:}  Coloured by H$_2$ to visualise
        mixing patterns.
  \item \textbf{Time-averaged fields:}  Use \texttt{H2Mean} for
        time-averaged concentration, \texttt{UMean} for mean
        velocity.
  \item \textbf{Wall $y^+$:}  Plot \texttt{yPlus} (requires the
        \texttt{yPlus} function object or post-processing utility).
\end{enumerate}


\section{Coefficient of Variation (COV)}

The COV quantifies mixing uniformity on a cross-section:
\[
  \text{COV} = \frac{\sigma_{\phi}}{\bar{\phi}}
\]

\noindent Compute from time-averaged fields:
\begin{itemize}
  \item $\bar{\phi}$ from \texttt{H2Mean}.
  \item $\sigma_{\phi}$ from
        $\sqrt{\texttt{H2Prime2Mean}}$.
\end{itemize}

\noindent $\text{COV} = 0$ indicates perfect mixing;
$\text{COV} = 1$ indicates complete segregation.

\chapter{Quick-Start Summary}
\label{ch:quickstart}

\noindent The complete workflow from a clean directory to a running
simulation:

\begin{enumerate}
  \item \textbf{Install dependencies:}
\begin{lstlisting}[language=bash,numbers=none]
pip install numpy numpy-stl
\end{lstlisting}

  \item \textbf{Generate STL geometry:}
\begin{lstlisting}[language=bash,numbers=none]
python3 generateSTL.py
\end{lstlisting}

  \item \textbf{Create background mesh:}
\begin{lstlisting}[language=bash,numbers=none]
blockMesh
\end{lstlisting}

  \item \textbf{Extract surface features:}
\begin{lstlisting}[language=bash,numbers=none]
surfaceFeatureExtract
\end{lstlisting}

  \item \textbf{Generate the final mesh:}
\begin{lstlisting}[language=bash,numbers=none]
snappyHexMesh -overwrite
\end{lstlisting}

  \item \textbf{Patch initial conditions:}
\begin{lstlisting}[language=bash,numbers=none]
setFields
\end{lstlisting}

  \item \textbf{Decompose for parallel run:}
\begin{lstlisting}[language=bash,numbers=none]
decomposePar
\end{lstlisting}

  \item \textbf{Run the simulation:}
\begin{lstlisting}[language=bash,numbers=none]
mpirun -np 8 reactingFoam -parallel
\end{lstlisting}

  \item \textbf{Reconstruct and post-process:}
\begin{lstlisting}[language=bash,numbers=none]
reconstructPar
paraFoam
\end{lstlisting}
\end{enumerate}

\noindent Or simply run everything with:

\begin{lstlisting}[language=bash,numbers=none]
./Allrun
\end{lstlisting}
